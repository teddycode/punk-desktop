### 核心 API 📚

```typescript
// 钱包服务
const wallet = useWalletService();

// 状态（响应式）
wallet.isConnected.value; // boolean
wallet.address.value; // string
wallet.chainId.value; // number

// 方法
await wallet.initWallet(callback, userInfo);
await wallet.openWallet();
await wallet.disconnectWallet();
wallet.checkInitialized();
wallet.getWalletInstance();
```

## 📝 使用示例

### 基础使用

```vue
<script setup>
import { inject } from 'vue';

const wallet = inject('wallet');

const handleConnect = async () => {
  await wallet.openWallet();
  console.log('已连接:', wallet.address.value);
};
</script>

<template>
  <button @click="handleConnect">
    {{ wallet.isConnected.value ? wallet.address.value : '连接钱包' }}
  </button>
</template>
```

### 在 Options API 中使用

```javascript
export default {
  methods: {
    async connectWallet() {
      await this.$wallet.openWallet();
    },
  },
};
```

### 在浏览器中使用

```javascript
import browser from '@js/common/browser';
browser.openInTable('https://app.uniswap.org', { wallet: true });
```

### 在 iframe 中使用

```javascript
import { WalletClient } from '@js/common/walletBridge';

const client = new WalletClient();
await client.connect();
const status = await client.getStatus();
```

## ⚠️ 注意事项

### 重要提醒

1. **单例模式**: 钱包服务全局只会初始化一次，不要重复创建
2. **初始化时机**: 必须在 Splash.vue 的 mounted 钩子中完成初始化
3. **错误处理**: 所有钱包操作都应该包裹在 try-catch 中
4. **用户体验**: 连接失败或超时时给予明确提示
5. **安全性**: iframe 通信时务必验证消息来源

### 已知限制

- TypeScript 类型定义可能需要根据实际情况调整
- 某些全局对象（如 `tsbApi`, `ipc`）的类型声明需要完善
- Web3Modal 对话框样式可以根据需求自定义

## 🐛 故障排除

### 常见问题

| 问题                              | 原因                         | 解决方案                        |
| --------------------------------- | ---------------------------- | ------------------------------- |
| `inject('wallet')` 返回 undefined | 插件未注册                   | 在 main.ts 中注册 walletPlugin  |
| 钱包未初始化                      | Splash.vue 未调用 initWallet | 检查 Splash.vue 的 mounted 方法 |
| iframe 无法访问钱包               | WalletBridge 未初始化        | 在父窗口初始化 WalletBridge     |
| 状态不持久化                      | Pinia 配置问题               | 检查 wallet.ts 的 persist 配置  |
| 连接超时                          | 网络或用户操作问题           | 增加超时时间或优化提示          |
