## 🔧 关键 Bug 修复历程

### Bug 1: 初始化时机问题

**现象**: `Uncaught Error: Please call "createWeb3Modal" before using "useWeb3ModalAccount"`

**原因**: 在插件注册时就调用了 `useWeb3ModalAccount()`，但此时 `createWeb3Modal` 还没执行。

**解决方案**:

- 使用动态 `import()` 代替 `require()`
- 延迟初始化 hooks，只在 `createWeb3Modal` 成功后才调用

### Bug 2: 响应式状态不更新

**现象**:

```
useWalletService: isConnected=true, address=0x..., chainId=11155111 ✅
Vue组件显示:    isConnected=false, address='', chainId=0       ❌
```

**原因**:

1. 每次调用 `useWalletService()` 都创建新的 computed
2. 这些 computed 虽然引用同一个 `accountRef`，但 computed 本身是新实例
3. Vue 的响应式系统无法追踪后来才赋值的 `accountRef`

**解决方案 - 三步架构改进**:

#### 步骤 1: 服务单例模式

```typescript
let serviceInstance = null;

export function useWalletService() {
  // 如果已创建，直接返回同一个实例
  if (serviceInstance) {
    return serviceInstance;
  }

  // 创建并缓存实例
  serviceInstance = { ... };
  return serviceInstance;
}
```

#### 步骤 2: 使用 ref 存储状态

```typescript
// 创建一个 ref 作为状态中间层
const walletState = ref({
  isConnected: false,
  address: '',
  chainId: 0,
});

// computed 基于这个 ref
const isConnected = computed(() => walletState.value.isConnected);
const address = computed(() => walletState.value.address);
const chainId = computed(() => walletState.value.chainId);
```

#### 步骤 3: 手动同步状态

```typescript
function syncWalletState() {
  if (!accountRef) return;

  // 将 accountRef 的值同步到 walletState
  walletState.value = {
    isConnected: accountRef.isConnected.value,
    address: accountRef.address.value,
    chainId: accountRef.chainId.value,
  };

  // 这会触发所有依赖 walletState 的 computed 重新计算！
}

// 在关键时刻调用
await initWallet(); // 初始化后调用
await openWallet(); // 连接成功后调用
await disconnectWallet(); // 断开后调用
```

## 📐 最终架构

### 响应式数据流

```
Web3Modal accountRef (外部)
         ↓
    syncWalletState()
         ↓
   walletState (ref)  ← 响应式源头
         ↓
   computed (isConnected, address, chainId)
         ↓
    serviceInstance (单例)
         ↓
   inject('wallet') (所有组件拿到同一个实例)
         ↓
    Vue UI 自动更新 ✅
```

### 关键设计原则

1. **单例模式**: 确保全局只有一个服务实例
2. **ref 中间层**: 使用 ref 作为可控的响应式源头
3. **手动同步**: 在状态变化时主动调用 `syncWalletState()`
4. **computed 依赖**: computed 依赖稳定的 ref，而不是动态变量

## 🎯 核心代码片段

### useWalletService.ts 核心结构

```typescript
import { ref, computed } from 'vue';

// 状态存储 (ref)
const walletState = ref({
  isConnected: false,
  address: '',
  chainId: 0,
});

// 单例缓存
let serviceInstance = null;
let accountRef = null;

// 状态同步函数
function syncWalletState() {
  if (!accountRef) return;
  walletState.value = {
    isConnected: accountRef.isConnected.value,
    address: accountRef.address.value,
    chainId: accountRef.chainId.value,
  };
}

export function useWalletService() {
  // 返回已有实例
  if (serviceInstance) {
    return serviceInstance;
  }

  // 基于 ref 的 computed
  const isConnected = computed(() => walletState.value.isConnected);
  const address = computed(() => walletState.value.address);
  const chainId = computed(() => walletState.value.chainId);

  const openWallet = async () => {
    // ... 连接逻辑
    syncWalletState(); // 成功后同步状态
  };

  // 创建并缓存实例
  serviceInstance = {
    isConnected,
    address,
    chainId,
    initWallet,
    openWallet,
    disconnectWallet,
    checkInitialized,
    getWalletInstance,
  };

  return serviceInstance;
}
```
