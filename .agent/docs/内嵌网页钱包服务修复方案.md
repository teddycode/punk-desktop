# 内嵌网页钱包服务修复方案

## 问题分析

### 核心问题
测试页面 `wallet-test.html` 在 BrowserView 内嵌窗口中打开时，无法访问钱包服务，显示：
- ❌ 环境检测：非 electron  
- ❌ 钱包服务：浏览器模式  
- ❌ 所有钱包操作失败

### 根本原因
1. **BrowserView 缺少 preload 脚本**
   - Splash.vue 中通过 `createWeb3Modal()` 创建的钱包服务只在桌面主窗口可用
   - BrowserView 是独立的渲染进程，没有注入 `window.wallet` 和 `window.ipc`
   - tableTabManager.js 创建 BrowserView 时 preload 被注释掉

2. **架构设计**
   ```
   桌面主窗口 (Splash.vue)
   ├─ Vue 应用 + Web3Modal 钱包服务
   └─ window.wallet 全局对象 ✅
   
   内嵌网页 (BrowserView)
   ├─ 独立渲染进程
   ├─ 无 preload 注入
   └─ window.wallet 不存在 ❌
   ```

## 解决方案

### 1. 创建钱包 Preload 脚本

**文件：** `d:\CodeProjects\Idea\punkos\js\preload\walletPreload.js`

**功能：**
- 通过 `contextBridge` 注入 `window.ipc` 接口
- 通过 `contextBridge` 注入 `window.wallet` 代理对象
- 钱包方法通过 IPC 与主进程通信，再转发到桌面钱包服务

**核心代码：**
```javascript
// 注入 IPC
contextBridge.exposeInMainWorld('ipc', {
  send: (channel, data) => ipcRenderer.send(channel, data),
  on: (channel, func) => ipcRenderer.on(channel, (...args) => func(...args)),
  invoke: (channel, ...args) => ipcRenderer.invoke(channel, ...args),
});

// 注入钱包代理
contextBridge.exposeInMainWorld('wallet', {
  async initWallet() {
    return await ipcRenderer.invoke('wallet-init');
  },
  async openWallet() {
    return await ipcRenderer.invoke('wallet-connect');
  },
  async disconnectWallet() {
    return await ipcRenderer.invoke('wallet-disconnect');
  },
  async getWalletState() {
    return await ipcRenderer.invoke('wallet-get-state');
  },
  // ... 其他方法
});
```

## 数据流

### 完整调用链路

```
内嵌网页 (wallet-test.html)
    ↓ window.wallet.openWallet()
    
Preload 脚本 (walletPreload.js)
    ↓ ipcRenderer.invoke('wallet-connect')
    
主进程 (tableTabManager.js)
    ↓ ipc.handle('wallet-connect')
    ↓ tableWin.webContents.send('wallet-connect-request')
    
桌面主窗口 (Splash.vue / Browser.vue)
    ↓ ipc.on('wallet-connect-request')
    ↓ useWalletService().openWallet()
    
Web3Modal 钱包服务
    ↓ 打开连接对话框
    ↓ 用户授权
    
返回结果
    ← webContents.send('wallet-connect-response', result)
    ← ipcRenderer.on('wallet-connect-response')
    ← 更新 window.wallet._address, _chainId
```

**问题：** 当前 IPC 处理器只是发送消息，没有实现真正的双向通信

#### 1. 在桌面主窗口添加 IPC 监听器
**文件：** `vite/packages/table/page/Browser/Index.vue` 或 `vite/packages/table/page/Splash.vue`

```vue
<script setup>
import { inject } from 'vue';

const wallet = inject('wallet'); // 获取钱包服务

// 监听内嵌网页的钱包请求
window.ipc.on('wallet-init-request', async (data) => {
  console.log('[Browser] 收到钱包初始化请求');
  try {
    const initialized = wallet.checkInitialized();
    // 发送结果回内嵌网页
    window.ipc.send('wallet-init-response', {
      success: true,
      initialized
    });
  } catch (error) {
    window.ipc.send('wallet-init-response', {
      success: false,
      error: error.message
    });
  }
});

window.ipc.on('wallet-connect-request', async (data) => {
  console.log('[Browser] 收到钱包连接请求');
  try {
    const account = await wallet.openWallet();
    window.ipc.send('wallet-connect-response', {
      success: true,
      address: wallet.address.value,
      chainId: wallet.chainId.value
    });
  } catch (error) {
    window.ipc.send('wallet-connect-response', {
      success: false,
      error: error.message
    });
  }
});

window.ipc.on('wallet-disconnect-request', async (data) => {
  console.log('[Browser] 收到钱包断开请求');
  try {
    await wallet.disconnectWallet();
    window.ipc.send('wallet-disconnect-response', {
      success: true
    });
  } catch (error) {
    window.ipc.send('wallet-disconnect-response', {
      success: false,
      error: error.message
    });
  }
});

window.ipc.on('wallet-get-state-request', async (data) => {
  console.log('[Browser] 收到获取钱包状态请求');
  window.ipc.send('wallet-get-state-response', {
    success: true,
    initialized: wallet.checkInitialized(),
    connected: wallet.isConnected.value,
    address: wallet.address.value,
    chainId: wallet.chainId.value
  });
});
</script>
```

#### 3. 在 Preload 中监听响应并更新状态
**文件：** `js/preload/walletPreload.js`

```javascript
// 监听响应并更新状态
ipcRenderer.on('wallet-connect-response', (event, result) => {
  if (result.success) {
    window.wallet._connected = true;
    window.wallet._address = result.address;
    window.wallet._chainId = result.chainId;
  }
  // 触发自定义事件通知页面
  window.dispatchEvent(new CustomEvent('wallet-connected', { detail: result }));
});

ipcRenderer.on('wallet-disconnect-response', (event, result) => {
  if (result.success) {
    window.wallet._connected = false;
    window.wallet._address = null;
    window.wallet._chainId = null;
  }
  window.dispatchEvent(new CustomEvent('wallet-disconnected', { detail: result }));
});
```

### 🟡 中等优先级

#### 1. Promise 化 IPC 调用
**问题：** 当前 `ipc.handle` 返回的是立即结果，实际需要等待桌面钱包操作完成

**解决：** 使用 Promise + 超时机制
```javascript
ipc.handle('wallet-connect', async (event) => {
  return new Promise((resolve, reject) => {
    const timeout = setTimeout(() => {
      reject(new Error('钱包连接超时'));
    }, 30000); // 30秒超时

    // 发送请求
    this.tableWin.webContents.send('wallet-connect-request', {
      requestId: Date.now() // 用于匹配请求和响应
    });

    // 临时监听器，接收响应
    const listener = (event, result) => {
      clearTimeout(timeout);
      ipcMain.off('wallet-connect-complete', listener);
      resolve(result);
    };
    ipcMain.once('wallet-connect-complete', listener);
  });
});
```

#### 2. 错误处理和重试机制
- 网络连接失败时自动重试
- 用户取消操作的友好提示
- 钱包未初始化时的自动初始化
